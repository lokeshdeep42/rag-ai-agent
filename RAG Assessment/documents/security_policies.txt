1. INFORMATION SECURITY OVERVIEW
=================================

1.1 Purpose
-----------
This document establishes the information security policies and procedures that all employees, contractors, and third parties must follow to protect company assets, data, and systems.

1.2 Scope
---------
These policies apply to:
- All company employees (full-time, part-time, contractors)
- All company-owned devices and systems
- All company data, regardless of location
- Third-party vendors with access to company systems

1.3 Compliance
--------------
Violation of these security policies may result in disciplinary action, up to and including termination of employment and legal action.

All employees must complete annual security awareness training and acknowledge understanding of these policies.

2. PASSWORD AND AUTHENTICATION POLICY
======================================

2.1 Password Requirements
--------------------------
All passwords must meet the following minimum requirements:

- Minimum length: 12 characters
- Must contain at least:
  * One uppercase letter (A-Z)
  * One lowercase letter (a-z)
  * One number (0-9)
  * One special character (!@#$%^&*)
- Cannot contain your name, username, or common words
- Cannot be a previously used password (last 12 passwords)
- Must be changed every 90 days

2.2 Password Management
------------------------
- Never share your password with anyone, including IT staff
- Do not write passwords down or store them in plain text
- Use a company-approved password manager (LastPass, 1Password)
- Do not use the same password across multiple systems
- Never send passwords via email or instant messaging

2.3 Multi-Factor Authentication (MFA)
--------------------------------------
MFA is REQUIRED for:
- All remote access (VPN, remote desktop)
- Email and collaboration tools (Office 365, Slack)
- Cloud services and SaaS applications
- Administrative access to any system
- Access to customer data or financial systems

Approved MFA methods:
- Authenticator apps (Microsoft Authenticator, Google Authenticator)
- Hardware security keys (YubiKey)
- SMS verification (least preferred, use only if other methods unavailable)

2.4 Account Lockout
-------------------
After 5 failed login attempts, accounts will be locked for 30 minutes. Contact IT Help Desk if you need immediate assistance.

3. DATA CLASSIFICATION AND HANDLING
====================================

3.1 Data Classification Levels
-------------------------------

PUBLIC
- Information intended for public consumption
- Marketing materials, press releases
- No special handling required

INTERNAL
- Information for internal use only
- Company policies, internal communications
- Should not be shared outside the company without approval

CONFIDENTIAL
- Sensitive business information
- Financial data, strategic plans, customer lists
- Requires encryption when stored or transmitted
- Access on need-to-know basis

RESTRICTED
- Highly sensitive information
- Customer personal data, trade secrets, source code
- Requires encryption and strict access controls
- Access requires executive approval

3.2 Data Handling Requirements
-------------------------------

Email Security:
- Use encryption for CONFIDENTIAL and RESTRICTED data
- Do not send sensitive data to personal email accounts
- Be cautious of phishing emails - verify sender before clicking links
- Report suspicious emails to security@company.com

File Storage:
- Store all work files on company-approved systems (OneDrive, SharePoint)
- Do not store company data on personal devices or cloud services
- CONFIDENTIAL and RESTRICTED data must be encrypted at rest
- Delete data when no longer needed

File Sharing:
- Use company-approved file sharing tools only
- Set appropriate permissions (view-only vs. edit)
- Set expiration dates on external shares
- Never share RESTRICTED data externally without approval

3.3 Data Retention and Disposal
--------------------------------
- Follow data retention schedules for your department
- Securely delete data when retention period expires
- Use secure deletion tools (not just "delete" key)
- Shred physical documents containing sensitive information
- Contact IT for secure disposal of storage devices

4. ACCEPTABLE USE POLICY
=========================

4.1 Company Equipment Use
--------------------------
Company-provided equipment (computers, phones, tablets) is for business use. Limited personal use is permitted but must not:
- Interfere with work responsibilities
- Consume excessive bandwidth or resources
- Violate any company policy
- Involve illegal activities

4.2 Prohibited Activities
--------------------------
The following activities are strictly prohibited:

- Installing unauthorized software or applications
- Accessing, storing, or transmitting illegal content
- Bypassing security controls or monitoring systems
- Using company resources for personal business ventures
- Cryptocurrency mining
- Torrenting or peer-to-peer file sharing
- Accessing inappropriate or offensive content
- Attempting to gain unauthorized access to systems

4.3 Internet and Email Use
---------------------------
- Internet access is monitored for security purposes
- Do not visit malicious or inappropriate websites
- Be professional in all email communications
- Company reserves the right to monitor email and internet usage
- No expectation of privacy on company systems

4.4 Software Installation
--------------------------
- Only IT-approved software may be installed
- Request software through the IT Service Portal
- Do not install browser extensions without approval
- Keep all software up to date with latest security patches

4.5 Personal Devices (BYOD)
----------------------------
If using personal devices for work:
- Must be enrolled in Mobile Device Management (MDM)
- Must have screen lock enabled (PIN, fingerprint, face ID)
- Must allow remote wipe capability
- Must keep OS and apps updated
- Company data will be wiped if device is lost or employment ends

5. INCIDENT RESPONSE
====================

5.1 Security Incident Reporting
--------------------------------
Report security incidents IMMEDIATELY to:
- Email: security@company.com
- Phone: (555) SEC-TEAM (24/7 hotline)
- IT Help Desk: helpdesk@company.com

Examples of security incidents:
- Lost or stolen devices
- Suspected malware infection
- Unauthorized access to systems or data
- Data breaches or leaks
- Phishing emails or social engineering attempts
- Suspicious system behavior

5.2 Response Time Requirements
-------------------------------
- Critical incidents: Report within 15 minutes
- High severity: Report within 1 hour
- Medium severity: Report within 4 hours
- Low severity: Report within 24 hours

5.3 What to Do If Compromised
------------------------------
If you suspect your account has been compromised:

1. Change your password immediately
2. Report to security team
3. Review recent account activity
4. Enable MFA if not already enabled
5. Monitor for suspicious activity

Do NOT:
- Try to "fix" the problem yourself
- Delete evidence
- Delay reporting

6. REMOTE ACCESS AND VPN
=========================

6.1 VPN Requirements
--------------------
All remote access to company systems MUST use the company VPN.

- VPN client: Cisco AnyConnect (download from IT portal)
- MFA is required for VPN connection
- VPN must be active when accessing company resources remotely
- Do not use third-party VPN services for company work

6.2 Remote Work Security
-------------------------
When working remotely:

- Use secure, password-protected Wi-Fi networks
- Never use public Wi-Fi without VPN
- Ensure physical security of devices
- Lock screen when away from device
- Do not allow others to view your screen
- Use privacy screen filters in public spaces

6.3 Home Network Security
--------------------------
Secure your home network:
- Change default router password
- Enable WPA3 or WPA2 encryption
- Keep router firmware updated
- Use a strong Wi-Fi password
- Disable WPS (Wi-Fi Protected Setup)
- Consider separate guest network for personal devices

7. PHYSICAL SECURITY
====================

7.1 Office Access
-----------------
- Access badges must be worn visibly at all times
- Do not tailgate or allow others to tailgate
- Report lost or stolen badges immediately
- Do not share your access badge
- Visitors must be escorted at all times

7.2 Clean Desk Policy
----------------------
At end of day:
- Lock away all confidential documents
- Log out of all systems
- Lock your workstation (Windows+L or Ctrl+Cmd+Q)
- Do not leave devices unattended
- Shred sensitive documents

7.3 Device Security
-------------------
- Enable full disk encryption on all devices
- Enable screen lock (maximum 5 minutes idle time)
- Use cable locks for laptops in office
- Never leave devices in vehicles
- Report lost or stolen devices within 1 hour

7.4 Visitor Management
----------------------
- All visitors must sign in at reception
- Visitors must wear visitor badges
- Employees are responsible for their visitors
- Do not discuss confidential information around visitors
- Escort visitors to/from meeting rooms
